# Work Order: make the envelope seams REAL interfaces

## Context
Every W3 WO was scoped away from `app/store/base.py` and the facade Protocols, so FIVE structural
workarounds accumulated: `_EnvelopeStore` (approval/envelope.py), `_EnvelopeSeamStore`
(reconciliation.py), `_EnvelopeStoreOps` (monitoring.py), `_EnvelopeFacadeOps`
(routes_trading.py), plus `EnvelopeTransitionError` living in store/core.py. Worse (CC-06):
several were applied via `cast(...)` — including `cast(Any, store)` at both production executor
call sites — which mypy never verifies, so "mypy green" proved nothing at exactly these seams.

## Scope (delivered)
1. Declared the full envelope API (create/get/list/transition/supersede/record_envelope_fill/
   approve_envelope_activation/stage_envelope_action) as abstract methods on `StateStore`
   (app/store/base.py); relocated `EnvelopeTransitionError` to base.py with a compat re-export
   from core.py. `get_position` was already on the ABC.
2. Extended the facade Protocols: `list_envelopes` on `ExecutionQueryFacade`;
   `approve_envelope` / `cancel_envelope` on `ExecutionCommandFacade`.
3. Deleted the four structural Protocols and every envelope-seam `cast`/`_cast` — direct typed
   calls throughout (approval gate, reconciliation executor, monitoring tick, trading routes,
   store-backed facade).
4. Proved `mypy app/` FAILS on a deliberate signature drift (drop AND mistype), then reverted.
