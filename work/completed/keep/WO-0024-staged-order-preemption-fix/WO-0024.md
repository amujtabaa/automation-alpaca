# Work Order: staged envelope order must never outlive envelope preemption — nor outlive its own validity

## Amendment (REV-0023 Phase A, pre-approval)

Three independent Phase A critics (INT-001, SPEC-03, CC-03) proved the original done-when
insufficient: the non-ACTIVE status guard closes the preemption hole but NOT the redrive hole —
in every new repro the envelope is still ACTIVE (fill-between-staging-and-redrive oversell,
post-TTL redrive, restart-redrive on zero market data). Scope widened to full redrive
re-validation. Authoritative: work/review/FINDING-W3-redrive-revalidation-bypass.md.

## Goal
A staged (CREATED, unexecuted) envelope order becomes structurally undrivable the moment it is no
longer valid to drive: (1) the venue leg re-reads the envelope and refuses+locally-cancels for
any non-ACTIVE envelope; (2) flatten/kill preemption cancels the preempted envelopes' CREATED
orders in the same atomic unit; (3) **redrive re-runs full write-time validation against CURRENT
envelope state and CURRENT time before the claim** — rails via the shared `validate_action`
(extended to cover TTL and allowed session phases, closing the "checked twice" gap on the fresh
path too), plus refusal when staged qty > remaining; refusal locally cancels the staged order
with envelope provenance. Flip the WO-0021 xfail to green (strict xfail will enforce).
