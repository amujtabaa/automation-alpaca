# S3 Reference (aws s3 / s3api)

All commands assume the session preamble ran (`$env:AWS_PAGER = ""`, identity verified) and use `--profile <YOUR_PROFILE> --region <YOUR_REGION> --output json`. Abbreviated below as `<P/R>`.

## High-level vs s3api

`aws s3` is the high-level porcelain (ls/cp/sync/rm/mb/rb/presign/website) with sane defaults and parallel transfers. `aws s3api` exposes the raw API one call per operation (bucket policy, public access block, lifecycle, versioned-object surgery). Prefer `aws s3` for transfers, `s3api` for configuration.

## Bucket lifecycle

| Operation           | Command                                 | Notes / Safety                                                 |
| ------------------- | --------------------------------------- | -------------------------------------------------------------- |
| List buckets        | `aws s3 ls <P/R>`                       | Read                                                           |
| Create bucket       | `aws s3 mb s3://<bucket> <P/R>`         | Mutation; names globally unique                                |
| Force-delete bucket | `aws s3 rb s3://<bucket> --force <P/R>` | GATE: irreversible; versioned buckets keep noncurrent versions |

## Object operations

| Operation                  | Command                                                                 | Notes / Safety                            |
| -------------------------- | ----------------------------------------------------------------------- | ----------------------------------------- |
| List objects (prefix)      | `aws s3 ls s3://<bucket>/path/ <P/R>`                                   | Read                                      |
| Copy file up               | `aws s3 cp localfile.txt s3://<bucket>/key <P/R>`                       | Mutation                                  |
| Copy with cache headers    | `aws s3 cp file.css s3://<bucket>/ --cache-control max-age=86400 <P/R>` | Mutation                                  |
| Copy directory recursively | `aws s3 cp ./dist/ s3://<bucket>/dist/ --recursive <P/R>`               | Mutation; confirm target prefix           |
| Delete object              | `aws s3 rm s3://<bucket>/path/file.txt <P/R>`                           | Mutation; irreversible without versioning |
| Delete prefix (recursive)  | `aws s3 rm s3://<bucket>/path/ --recursive <P/R>`                       | GATE: confirm prefix scope                |

## Sync and the --delete danger

| Operation                 | Command                                             | Notes / Safety                                              |
| ------------------------- | --------------------------------------------------- | ----------------------------------------------------------- |
| Sync (upload new/changed) | `aws s3 sync ./build s3://<bucket>/ <P/R>`          | Mutation; only adds/overwrites                              |
| Sync dry-run preview      | `aws s3 sync ./build s3://<bucket>/ --dryrun <P/R>` | Read; always before --delete                                |
| Sync with delete          | `aws s3 sync ./build s3://<bucket>/ --delete <P/R>` | GATE: removes remote objects absent locally; --dryrun first |

`--delete` is the single most dangerous routine flag in daily S3 work: a wrong local path silently deletes the remote tree. ALWAYS run the identical command with `--dryrun` first and read the planned deletions.

## Presigned URLs

| Operation             | Command                                                           | Notes / Safety                               |
| --------------------- | ----------------------------------------------------------------- | -------------------------------------------- |
| Presign (1h default)  | `aws s3 presign s3://<bucket>/file.pdf <P/R>`                     | Read; SigV4 only in v2; max 7 days (604800s) |
| Presign custom expiry | `aws s3 presign s3://<bucket>/file.pdf --expires-in 604800 <P/R>` | Read                                         |

Presigning needs no extra IAM action; the URL inherits the signer's permissions, so the profile must hold `s3:GetObject` when the link is used.

## Bucket policy and public access

| Operation               | Command                                                                                                                                                                                        | Notes / Safety                |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------- |
| Get bucket policy       | `aws s3api get-bucket-policy --bucket <bucket> <P/R>`                                                                                                                                          | Read                          |
| Put bucket policy       | `aws s3api put-bucket-policy --bucket <bucket> --policy file://$env:TEMP/policy.json <P/R>`                                                                                                    | GATE; always file://          |
| Delete bucket policy    | `aws s3api delete-bucket-policy --bucket <bucket> <P/R>`                                                                                                                                       | GATE                          |
| Block all public access | `aws s3api put-public-access-block --bucket <bucket> --public-access-block-configuration "BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true" <P/R>` | Mutation; recommended default |

## Safety notes

- `rb --force` on versioned buckets leaves delete markers/noncurrent versions behind (purge via an `s3api delete-objects` loop over `list-object-versions`).
- v2 presign is SigV4-only with a hard 7-day maximum.
- CRC64NVME is the default checksum since v2.23.0; do not mix `--checksum-algorithm` with old scripts.

## IAM actions the profile needs per task

- Read/audit: `s3:ListAllMyBuckets`, `s3:ListBucket`, `s3:GetObject`, `s3:GetBucketPolicy`, `s3:GetLifecycleConfiguration`
- Deploy/sync: + `s3:PutObject`, `s3:DeleteObject`
- Bucket admin: + `s3:CreateBucket`, `s3:DeleteBucket`, `s3:PutBucketPolicy`, `s3:DeleteBucketPolicy`, `s3:PutBucketPublicAccessBlock`, `s3:PutLifecycleConfiguration`, `s3:PutBucketWebsite`
